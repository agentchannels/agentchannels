# Contributing to agentchannels

Thanks for your interest in contributing! This guide covers how to set up a development environment, run tests, and submit changes.

## Prerequisites

- **Node.js** >= 18
- **pnpm** (10.7+) — `npm install -g pnpm`
- A **Slack workspace** where you can create test apps (for e2e tests)
- An **Anthropic API key** with Managed Agents access (for recording e2e fixtures)

## Setup

```bash
git clone https://github.com/agentchannels/agentchannels.git
cd agentchannels
pnpm install
```

## Development

```bash
# Type-check
pnpm lint

# Build (TypeScript -> dist/)
pnpm build

# Dev mode with hot reload (no build needed)
pnpm dev -- serve

# Run compiled
pnpm start
```

## Testing

### Unit tests

```bash
pnpm test              # Run all tests once
pnpm test:watch        # Watch mode

# Run a single test file
npx vitest run tests/core/chunk-parser.test.ts

# Run tests matching a pattern
npx vitest run -t "parseSSEEvent"
```

Unit tests mock all external dependencies (Slack API, Anthropic SDK) and don't require any env vars or network access.

### E2E tests

The e2e suite (`tests/e2e/`) runs against a **real Slack workspace** with a record/replay cassette pattern for the Claude Managed Agent.

#### Required env vars

Add these to your `.env` file:

| Variable | Description |
|---|---|
| `SLACK_TEST_USER_TOKEN` | `xoxp-...` user OAuth token — posts messages as you |
| `SLACK_TEST_CHANNEL_ID` | Dedicated test channel ID (e.g. `C0123456789`) |
| `SLACK_BOT_TOKEN` | `xoxb-...` bot token (same as `ach serve` uses) |
| `SLACK_APP_TOKEN` | `xapp-...` app-level token |

For recording fixtures (first run or re-record):

| Variable | Description |
|---|---|
| `ANTHROPIC_API_KEY` | Anthropic API key |
| `CLAUDE_AGENT_ID` | Claude Managed Agent ID |
| `CLAUDE_ENVIRONMENT_ID` | Claude Environment ID |
| `CLAUDE_VAULT_IDS` | Comma-separated vault IDs (optional) |

#### Getting a user token

1. Open your Slack app at `https://api.slack.com/apps/{appId}/oauth`
2. Under **User Token Scopes**, add: `chat:write`, `channels:history`, `groups:history`, `im:history`
3. **Reinstall to Workspace** and copy the **User OAuth Token** (`xoxp-...`)

#### Getting the test channel ID

1. Create a channel (e.g. `#agentchannels-e2e`) and invite your bot
2. Right-click the channel name -> Copy link -> the trailing `C0123456789` is the ID

#### Running e2e tests

```bash
# First run — records a fixture from the live agent
pnpm vitest run tests/e2e/slack-bridge-e2e.test.ts

# Subsequent runs — replays the fixture (no Anthropic API call)
pnpm vitest run tests/e2e/slack-bridge-e2e.test.ts

# Force re-record
E2E_RERECORD=true pnpm vitest run tests/e2e/slack-bridge-e2e.test.ts
```

When e2e env vars are missing, the suite **skips gracefully** — `pnpm test` always passes without them.

Fixtures are stored in `tests/e2e/fixtures/` (gitignored). Each contributor records their own on first run. Test messages are left in the dedicated test channel with unique tags per run — no cleanup needed.

## Code style

- **ES Modules** — all imports use `.js` extensions
- **TypeScript** — strict mode, no `any` where avoidable
- **Vitest** — `globals: false` (explicit `import { describe, it, expect } from "vitest"`)
- **No `.env` in repo** — generated by `ach init slack` / `ach init agent`

## Submitting changes

1. Fork the repo and create a branch from `main`
2. Make your changes
3. Run `pnpm lint && pnpm test` to verify
4. Commit with a clear message explaining the *why*, not just the *what*
5. Open a pull request against `main`

## Project structure

```
src/
  cli/          CLI entrypoint (commander)
  commands/     serve, init-agent
  channels/     Channel adapters (Slack)
  core/         StreamingBridge, AgentClient, SessionManager, config
  config/       Env file utilities
  deploy/       Railway deployment
tests/
  commands/     Unit tests for commands
  core/         Unit tests for core modules
  channels/     Unit tests for channel adapters
  e2e/          End-to-end tests (live Slack + cassette replay)
    helpers/    Test utilities, adapters, fixture loaders
    fixtures/   Recorded cassettes (gitignored)
```
