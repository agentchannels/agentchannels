## Project Overview

agentchannels (`ach`) is a TypeScript CLI tool that bridges messaging platforms to Claude Managed Agent sessions. Each Slack thread becomes a multi-turn streaming conversation with a Claude agent via Socket Mode (no public URL needed).

## Commands

```bash
pnpm install          # Install dependencies
pnpm build            # Compile TypeScript (tsc ‚Üí dist/)
pnpm dev -- serve     # Dev mode with tsx (no build needed)
pnpm start            # Run compiled: node dist/cli/index.js serve
pnpm test             # Run all tests (vitest run)
pnpm test:watch       # Watch mode
pnpm lint             # Type-check only (tsc --noEmit)

# Run a single test file
npx vitest run tests/core/chunk-parser.test.ts

# Run tests matching a pattern
npx vitest run -t "parseSSEEvent"
```

## Architecture

The core abstraction is the **channel adapter pattern** ‚Äî a `ChannelAdapter` interface that decouples messaging platforms from the agent-bridging logic.

### Message flow

```
Slack event ‚Üí SlackAdapter.onMessage() ‚Üí StreamingBridge.handleMessage()
  ‚Üí SessionManager (thread‚Üísession map) ‚Üí AgentClient.sendMessage() (SSE stream)
  ‚Üí SessionOutputReader (parses SSE events) ‚Üí StreamHandle (streams back to Slack)
```

### Key modules

- **`src/core/channel-adapter.ts`** ‚Äî `ChannelAdapter` interface, `ChannelMessage`, `StreamHandle` types. All channel implementations conform to this contract.
- **`src/core/streaming-bridge.ts`** ‚Äî Channel-agnostic coordinator. Connects `SessionOutputReader` to a `StreamHandle`. Manages per-thread `AbortController`, concurrency guards, and lifecycle phases.
- **`src/core/agent-client.ts`** ‚Äî Wraps `@anthropic-ai/sdk` beta API (`client.beta.agents`, `client.beta.sessions`, `client.beta.environments`). The `sendMessage()` method is an `AsyncGenerator<AgentStreamEvent>`.
- **`src/core/session-output-reader.ts`** ‚Äî Consumes `AgentClient.sendMessage()` generator, adds retry logic with exponential backoff, exposes both `EventEmitter` and `AsyncIterator` interfaces.
- **`src/core/chunk-parser.ts`** ‚Äî Pure function `parseSSEEvent()` that transforms raw SSE events into a discriminated union (`AgentStreamEvent`: text_delta, tool_use, tool_result, thinking, status, done, error).
- **`src/core/session-manager.ts`** ‚Äî In-memory `Map<threadKey, sessionId>` with optional TTL expiry. Key format: `"channelType:channelId:threadId"`.
- **`src/core/config.ts`** ‚Äî Three-source config resolution: CLI flags > env vars > .env file. Uses Zod validation.
- **`src/channels/slack/adapter.ts`** ‚Äî `SlackAdapter` implements `ChannelAdapter` using `@slack/bolt` Socket Mode. Uses Slack's `chat.startStream`/`appendStream`/`stopStream` for real-time streaming.
- **`src/commands/serve.ts`** ‚Äî Wires everything together: config ‚Üí AgentClient ‚Üí SessionManager ‚Üí SlackAdapter ‚Üí StreamingBridge.

### Config resolution

Config values are resolved in this precedence order (highest wins):
1. CLI flags (`--anthropic-api-key`)
2. `process.env`
3. `.env` file in cwd

The `resolveConfig()` function validates all required fields; `resolvePartialConfig()` is used by init commands that don't need full config.

## Conventions

- **ES Modules** ‚Äî `"type": "module"` in package.json; all imports use `.js` extension
- **pnpm** ‚Äî Package manager (pnpm@10.7.0)
- **Vitest** ‚Äî Tests in `tests/` mirror `src/` structure; `globals: false` (explicit imports)
- **Zod v4** ‚Äî Runtime validation for config schemas
- **No .env in repo** ‚Äî Generated by `ach init slack` and `ach init agent` wizards
- Tests mock external dependencies (Slack API, Anthropic SDK) ‚Äî no live API calls in tests
